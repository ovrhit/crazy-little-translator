# 프로젝트 규칙 (RULES.md)

이 문서는 '실시간 오버레이 번역기 앱' 개발 시 준수해야 할 핵심 원칙을 정의합니다.

## 1. 성능 및 최적화 원칙 (가장 중요)

- **비트맵 관리:** 모든 비트맵 처리는 메모리 누수를 방지하기 위해 사용 후 즉시 `recycle()` 하거나 전용 풀에서 관리할 것.
- **인식 결과 캐싱:** OCR 인식 결과가 이전 프레임과 동일할 경우, 불필요한 UI 재렌더링과 API 호출을 엄격히 금지함.
- **비동기 처리:** 화면 캡처 루프는 반드시 `HandlerThread`나 Coroutine의 특정 `Dispatcher`에서 실행하여 UI 스레드 부하를 최소화할 것.

## 2. 코드 스타일 및 아키텍처

- **아키텍처:** 안드로이드 최신 권장 아키텍처(MVVM)를 준수함.
- **UI 프레임워크:** UI 로직은 Jetpack Compose로만 작성함.
- **보안 및 환경 설정:** 하드코딩된 문자열이나 API 키는 별도의 `Secret.kt`나 `local.properties`에서 관리하도록 강제함.

## 3. 캐릭터 페르소나 관리

- **영속성:** 말투(페르소나) 프롬프트는 나중에 사용자가 수정할 수 있도록 `DataStore`나 `Database`에 저장하고 관리하는 구조를 유지할 것.

## 4. 예외 처리 규칙

- **사용자 알림:** 네트워크 단절이나 API 할당량 초과 시, 사용자에게 오버레이 알림으로 명확히 상태를 알릴 것.
